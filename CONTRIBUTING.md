# Contributing

Thanks for helping make this tiny project more fun, useful, and beginner-friendly. Bug fixes, accessibility improvements, clearer docs, and creative themes are all welcome.

## Make your own version

1. Select **Fork** on GitHub to create a copy under your account.
2. Clone your fork to your computer.
3. Create a branch for your changes:

   ```bash
   git checkout -b my-new-theme
   ```

4. Customize `index.html` and preview it in a browser.
5. Commit and push your version to your fork.

You do not need to submit a pull request if you only want a personalized template. Keep your fork, publish it, and make it yours.

## Submit an improvement

1. Check existing issues and pull requests to avoid duplicate work.
2. Create a focused branch from the latest default branch.
3. Make one clear improvement and test it on desktop and mobile when relevant.
4. Update documentation when behavior or customization steps change.
5. Open a pull request explaining what changed and why.

Please avoid unrelated rewrites or adding dependencies for features that plain HTML, CSS, and JavaScript can handle simply.

## Suggest a theme

Open a GitHub issue with a short description of the theme, suggested colors or mood, and any accessibility considerations. Examples might include friendship, birthday, pastel, dark mode, or pride themes. Do not include another person's private information or copyrighted assets you cannot share.

## Respect and safety

- Be kind and constructive in issues, reviews, and pull requests.
- Do not add harassment, manipulation, hateful content, explicit material, or code intended to invade privacy.
- Never commit secrets, API keys, phone numbers, addresses, private messages, or personal details without consent.
- Use images, music, fonts, and other assets only when their licenses allow redistribution.
- Respect a person's answer and boundaries; the project is a playful template, not a tool for pressuring anyone.
